3. 🗄️ Data Layer Design
3.1 LedgerEntry Table
CREATE TABLE ledger_entries (
  id SERIAL PRIMARY KEY,
  merchant_id INT NOT NULL,
  type VARCHAR(10) CHECK (type IN ('CREDIT','DEBIT')),
  amount_paise BIGINT NOT NULL,
  reference TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
3.2 Payout Table
CREATE TABLE payouts (
  id SERIAL PRIMARY KEY,
  merchant_id INT NOT NULL,
  amount_paise BIGINT NOT NULL,
  status VARCHAR(20),
  idempotency_key UUID,
  attempts INT DEFAULT 0,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
3.3 Idempotency Table
CREATE TABLE idempotency_keys (
  id SERIAL PRIMARY KEY,
  merchant_id INT,
  key UUID,
  response JSONB,
  created_at TIMESTAMP
);
3.4 Indexing Strategy
CREATE INDEX idx_ledger_merchant ON ledger_entries(merchant_id);
CREATE UNIQUE INDEX idx_idem_key ON idempotency_keys(merchant_id, key);
CREATE INDEX idx_payout_status ON payouts(status);
4. 🧠 Core Logic Design
4.1 Balance Calculation (DB-Level)
SELECT 
COALESCE(SUM(CASE WHEN type='CREDIT' THEN amount_paise END),0) -
COALESCE(SUM(CASE WHEN type='DEBIT' THEN amount_paise END),0)
FROM ledger_entries
WHERE merchant_id = %s;
4.2 Transaction Boundary

All payout creation must run inside:

@transaction.atomic
4.3 Concurrency Control
Mechanism:
Row-level locking
LedgerEntry.objects.select_for_update().filter(merchant_id=merchant_id)
Flow:
Begin transaction
Lock ledger rows
Compute balance
Validate amount
Insert DEBIT entry
Create payout
Commit
4.4 Idempotency Implementation
Step Flow:
1. Check idempotency_keys table
2. If exists → return stored response
3. Else:
   - process request
   - store response
   - return response
DB Constraint:
UNIQUE (merchant_id, key)
Edge Case Handling:
If two requests arrive simultaneously:
Unique constraint ensures only one insert succeeds
Second request fetches stored response
5. ⚙️ Payout Engine
5.1 State Machine (Strict Enforcement)
pending → processing → completed
pending → processing → failed
Validation Logic:
if current_status == "completed":
    raise Exception("Invalid transition")
5.2 Async Job Execution
Trigger:
async_task("app.tasks.process_payout", payout_id)
5.3 Worker Logic
Function:
def process_payout(payout_id):
    payout = get_payout()

    move_to_processing()

    simulate_result()

    if success:
        mark_completed()
    elif fail:
        mark_failed()
        refund()
